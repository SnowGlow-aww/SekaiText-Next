# 常见问题

## 安装与启动

### macOS 提示「已损坏，无法打开」/「无法验证开发者」

未签名应用的正常放行流程：右键 → 打开；仍不行时在终端执行后重开：

```bash
sudo xattr -cr "/Applications/SekaiText Next.app"
```

### Windows SmartScreen 拦截

点「更多信息」→「仍要运行」。

## 下载与更新

### 下载 / 更新速度慢？

应用内所有下载（更新包、插件、Live2D 资源、剧情目录元数据）已全部走国内 CDN 加速，正常无需代理。如仍异常缓慢，检查本地网络或稍后重试。

### 启动时「拉取元数据」卡住不动？

升级到 **v5.7.1 及以上**。旧版本元数据直连海外数据源，慢网络下会卡死；新版本走 CDN 镜像并实时校验一致性，全量拉取一般在 10 秒内完成。

### 自动更新失败

到 [下载页](../download) 手动下载最新安装包覆盖安装即可，数据不会丢失。

## 功能

### 语音点了没声音？

在设置中切换语音源后重试——部分数据源不托管语音文件。

### Live2D 同步总提示有资源缺失

先重试一次同步（已支持自动回退备用源）。若固定提示同样的失败数，通常是个别资源上游缺失，不影响其他剧情正常播放；请保持应用为最新版本。

### 打轴匹配结果为 0？

基本都是视频和剧情选错对应关系——确认录屏与所选剧情是同一期活动/同一话（注意脚本编号与活动实装编号可能有偏移）。

### Windows 压制启动就失败（`Unknown encoder` / `Header too large`）？

5.7.2 及更早版本的已知问题：压制编码器默认了 macOS 专属项，装过老版 SekaiTools 的机器还会被残留组件抢走压制管线。升级 SekaiText 到 **5.7.3** 并在插件市场把「自动轴机 + 压制」更到 **3.1.0** 即可——编码器现在按显卡自动判定（NVIDIA / Intel / AMD 硬编逐个验证后才列出），无硬编时自动用 x264 软编。

### 压制进度一直显示 0%？

5.8.4 及更早版本在部分 Windows 机器上拿不到 FFmpeg 状态行，个别机器还伴随视频探测挂起。5.8.5 起加入机读进度通道；建议直接升级到 **5.8.13**，同时获得最新的静默环境保护，插件无需更新。

若日志显示「启动 25 秒无任何进度输出且输出文件无增长」，通常是显卡驱动的硬解管线起步挂起：程序会自动关闭硬解重试，也可以在压制前手动关闭「硬解」。如果 QSV、甚至自动降级后的 x264 已经输出几十或上百 MB，却在「watchdog 静默模式」等待 120 秒后被终止，则是 5.8.12 及更早版本的误判；5.8.13 起，只要确认产生过有效媒体数据就会继续等待，不再凭进度静默或暂时无落盘强杀任务。反馈问题时请附上自动导出的压制日志，其中包含系统、显卡驱动与 FFmpeg 版本。

## 其他

### 并行压制概率性失败（`ffmpeg 退出码 -19` 等）？

多个压制任务并发时，硬件编码会话可能超出显卡驱动的额度（AMD/AMF 表现为退出码 -19，NVIDIA 表现为 -12）。升级 SekaiText 到 **5.7.4** 并把「自动轴机 + 压制」插件更到 **3.2.0**：硬件编码器启动失败会自动降级 x264 软编重跑，任务不再挂掉；压制报错时日志会自动导出到数据目录 `logs/`，反馈问题时附上即可。

### 译文保存在哪里？会丢吗？

自 v5.7.0 起每次编辑约 1 秒后自动写入正式文档（默认 `~/文稿/SekaiText/故事类型/索引/` 下按规范命名，可在 设置 → 文稿保存路径 更换位置，已有文稿自动迁移）；应用数据目录另有每 30 秒一份的恢复文件，崩溃断电后启动会提示恢复。卸载重装不清除数据；重要项目仍建议定期备份。

### 更新插件后界面没变化？

把插件禁用再启用一次（或重启应用），热加载的旧实例即被替换。

### 遇到 bug 怎么反馈？

在设置中打开 **调试日志**，复现问题后带着导出的日志到 [GitHub Issues](https://github.com/SnowGlow-aww/SekaiText-Next/issues) 反馈。
