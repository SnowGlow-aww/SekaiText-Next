# 自动打轴与压制

SekaiText Next 内置 **SekaiCoreEngine** 打轴内核：识别游戏录屏中的对话画面，把翻译好的每句台词自动对齐到视频时间轴；打完轴可以直接在行列表里逐行微调分句，与 Aegisub 双向同步精修，最后经 FFmpeg 一键压制输出成品。

::: tip 版本要求
本页描述的行列表 / 分句微调 / Aegisub 同步功能需要 **SekaiText Next ≥ 5.2.0** 且「自动轴机 + 压制」插件 **≥ 2.0.0**；**并行任务 / 语音停顿对齐 / 译文回读**需要 **SekaiText Next ≥ 5.5.0** 且插件 **≥ 2.2.0**；**staff 制作人员行 / 阈值与 staff 预设 / 横幅时间精修**需要 **SekaiText Next ≥ 5.7.0** 且插件 **≥ 3.0.0**；**压制编码器按显卡自动判定（Windows 压制修复）**需要 **SekaiText Next ≥ 5.7.3** 且插件 **≥ 3.1.0**（插件市场直接更新即可；更新后若界面没变化，把插件禁用再启用一次）。
:::

## 准备工作

1. **游戏录屏**：完整录制目标剧情的播放过程（对话 auto 即可）；
2. **完成翻译**：确保该篇剧情的译文已在编辑器中完成。

## 打轴流程

界面为双列布局：**左列**选文件、启动识别、看画面预览；**右列**是识别出的行列表。

1. 在左列选择 **视频文件** 与 **对应剧情**，开始识别 — 引擎逐帧分析画面上的对话文本，与剧本逐句匹配；
2. 识别过程中右列就会陆续列出已匹配的行；完成后即可逐行检查、微调；
3. 确认无误后点 **导出 ass**（不再自动导出——先调完再导，避免反复覆盖）。

::: warning 匹配结果为 0 或大量错位？
几乎都是 **视频和剧本对不上**：确认选择的剧情与录屏是同一期活动 / 同一话。注意剧情脚本编号与活动实装编号可能存在偏移，错选就会出现「识别失效」的情况。
:::

## 并行任务：同时处理多个视频

需要连打多期剧情时，勾选左列的 **并行任务模式**，就可以在上一个视频还在打轴 / 压制时直接换文件启动下一个：

- 打轴与压制界面各自出现**任务列表**，显示每个任务的进度；点击卡片切换查看，运行中的可单独取消，完成的点 **✕** 关闭；
- 每个并行任务独占一套识别 / 编码内核进程，CPU 与内存开销随任务数成倍增长——**性能不高的电脑请慎用**（同类任务最多并行 4 个）；
- 完成的任务请及时关闭以释放内存；
- 离开页面再回来（甚至插件重载后），仍在后台运行的任务会自动找回。

不开启开关时行为与从前完全一致：同一时刻一个打轴 + 一个压制。

## 行列表与分句微调

译文超长的行（原文三行、译文超过阈值）会被标为 **过长**，需要拆成前后两条字幕分批显示。工具栏勾选「仅显示过长行」可以只看这些行。点开任意对话行即可微调：

- **文本分割点** — 拖动滑块决定前后两句在哪个字断开，预览直接显示两半内容；
- **换行时机** — 决定第几帧切换到后半句。支持滑块、直接输入帧号、`±1 / ±5` 逐帧微调；
- **建议帧** — 按游戏打字机速度（80ms/字）自动估算换行时刻，一键应用；
- **按字数均分** — 无配音台词按前后字数比例均分显示时长；
- **语音停顿** — 拉取该句语音做静音检测，把说话人语句之间的停顿列成候选换行点，点击即应用——有配音的台词按实际说话节奏分句最自然（语音走临时缓存，软件重启即清空）；
- **画面预览** — 实时显示换行帧那一刻的画面，不用去视频里来回拖进度条；
- **溢出预警** — 某半句可能在打字机打完前就被切走时会提示，适当移动分割点或换行时机即可。

双击译文文本可以就地快速修改（最多两行）。

## 与 Aegisub 双向同步

导出时默认写入同步标识（Effect 字段 `st:N`），从此不用在轴机和 Aegisub 之间来回导入导出：

- **Aegisub → 轴机**：在 Aegisub 里精修后直接 `Ctrl+S` 保存，轴机检测到文件变化会**自动回读译文与换行时间**（在 Aegisub 里改的字不会再被下次导出覆盖），行列表跟着刷新；也可以随时点 **从 Aegisub 拉取** 手动回读；
- **轴机 → Aegisub**：在轴机里改过的行点 **推送到 Aegisub**，然后在 Aegisub 运行「自动化 → SekaiText → 从轴机拉取」，改动会带撤销点地原位替换进当前字幕。

同步宏无需手动安装：导出时若检测到本机装有 Aegisub，会自动装进其 `automation/autoload` 目录（首次使用重启一次 Aegisub 即可）。**便携版 Aegisub** 的配置跟着程序走、自动探测不到时，在「导出与同步选项」里手动指定 `automation/autoload` 目录并点 **安装宏** 即可，一次搞定。

## 成品样式与导出

导出默认就是成品：内置团队样式模板（1行 / 2行 / 3行 等定义）自动套用，按每行分行数改样式（`\N` 换行完整保留）、清理角色名与调试行；**地点横幅**也会自动改用「遮罩 / 地点名称」样式，时间轴按画面中灰色横条的出现 / 消失逐帧对齐。有自定义需求时，在「导出与同步选项」里选择自己的模板 `.ass` 即可覆盖内置模板。

另外，每次分句 / 译文微调后都会自动把当前完整字幕写到输出目录的 `autosave-<剧本名>.ass` 保险文件——崩溃误退也不丢微调。

### staff 制作人员行 <Badge type="tip" text="5.7.0+" />

「导出与同步选项」里可以填写 staff 表（录制 / 翻译 / 校对 / 时轴 / 轴校&压制等），导出时自动作为第一条字幕注入成品（0:00 起 5 秒淡入淡出）：

- 「」内的人名全部自定义，留空的职位自动跳过；
- 时轴和「轴校&压制」是同一人时，自动合并为 `时轴&轴校&压制：「某某」`；
- 填写内容本机记忆，也可以存成**命名预设**一键切换（识别阈值同样支持预设）。

### 横幅时间精修 <Badge type="tip" text="5.7.0+" />

地点横幅的出现 / 消失时间在 5.7.0 起做了帧级精修：识别淡入起笔与淡出收尾（对人工校对基准误差 ≤2 帧，且刻意偏早半帧量级——宁可早现绝不露出日文原字）。无需任何配置，重新打轴即生效。

## 一键压制

打轴完成后可直接压制（在页面下方）：内置 FFmpeg + libass，将字幕烧录进视频输出成品文件。

- **编码器按显卡自动判定**（5.7.3 起）：进入页面时对本机每个硬件编码器（macOS VideoToolbox / NVIDIA NVENC / Intel QSV / AMD AMF）做真实试编码验证，下拉框只列出确认可用的项并自动选中推荐的硬编；没有可用硬编时回落 x264 软编，保证一定能压。手动换过的选择会记住；
- 支持硬件加速，实测压制速度可提升 4～7 倍；
- 随引擎附带字幕默认字体，未装字体的机器也能压出正确字形。

## 常见问题

- **识别很慢**：识别为逐帧分析，时长与视频长度、机器性能相关，属正常现象；
- **行列表提示需要更新主程序**：插件是新版但 SekaiText 本体还是旧版，升级主程序到 5.2.0 以上即可；
- **Windows 压制启动就失败 / 报 `Unknown encoder` 或 `Header too large`**：这是 5.7.2 及更早版本的已知问题（编码器默认了 macOS 专属项；装过老版 SekaiTools 的机器还会被残留的 VapourSynth 抢走压制管线）——升级 SekaiText 到 **5.7.3** 并把插件更到 **3.1.0** 即可，压制已改走自带 FFmpeg 且编码器按显卡自动判定；
- **导出的 ass 样式和团队成品不一致**：确认没有在导出选项里误选了别的模板文件（留空即用内置模板）；
- **插件市场更新插件后界面没变化**：已加载的旧版本还在内存里——在插件市场把该插件禁用再启用（或重启软件）即可。
