# 自动打轴与压制

SekaiText Next 内置 **SekaiCoreEngine** 打轴内核：识别游戏录屏中的对话画面，把翻译好的每句台词自动对齐到视频时间轴；打完轴可以直接在行列表里逐行微调分句，与 Aegisub 双向同步精修，最后经 FFmpeg 一键压制输出成品。

::: tip 版本要求
本页描述的行列表 / 分句微调 / Aegisub 同步功能需要 **SekaiText Next ≥ 5.2.0** 且「自动轴机 + 压制」插件 **≥ 2.0.0**（插件市场直接更新即可）。
:::

## 准备工作

1. **游戏录屏**：完整录制目标剧情的播放过程（对话 auto 即可）；
2. **完成翻译**：确保该篇剧情的译文已在编辑器中完成。

## 打轴流程

界面为双列布局：**左列**选文件、启动识别、看画面预览；**右列**是识别出的行列表。

1. 在左列选择 **视频文件** 与 **对应剧情**，开始识别 — 引擎逐帧分析画面上的对话文本，与剧本逐句匹配；
2. 识别过程中右列就会陆续列出已匹配的行；完成后即可逐行检查、微调；
3. 确认无误后点 **导出 ass**（不再自动导出——先调完再导，避免反复覆盖）。

::: warning 匹配结果为 0 或大量错位？
几乎都是 **视频和剧本对不上**：确认选择的剧情与录屏是同一期活动 / 同一话。注意剧情脚本编号与活动实装编号可能存在偏移，错选就会出现「识别失效」的情况。
:::

## 行列表与分句微调

译文超长的行（原文三行、译文超过阈值）会被标为 **过长**，需要拆成前后两条字幕分批显示。工具栏勾选「仅显示过长行」可以只看这些行。点开任意对话行即可微调：

- **文本分割点** — 拖动滑块决定前后两句在哪个字断开，预览直接显示两半内容；
- **换行时机** — 决定第几帧切换到后半句。支持滑块、直接输入帧号、`±1 / ±5` 逐帧微调；
- **建议帧** — 按游戏打字机速度（80ms/字）自动估算换行时刻，一键应用；
- **按字数均分** — 无配音台词按前后字数比例均分显示时长；
- **画面预览** — 实时显示换行帧那一刻的画面，不用去视频里来回拖进度条；
- **溢出预警** — 某半句可能在打字机打完前就被切走时会提示，适当移动分割点或换行时机即可。

双击译文文本可以就地快速修改（最多两行）。

## 与 Aegisub 双向同步

导出时默认写入同步标识（Effect 字段 `st:N`），从此不用在轴机和 Aegisub 之间来回导入导出：

- **Aegisub → 轴机**：在 Aegisub 里精修后直接 `Ctrl+S` 保存，轴机检测到文件变化会**自动回读**换行时间，行列表跟着刷新；
- **轴机 → Aegisub**：在轴机里改过的行点 **推送到 Aegisub**，然后在 Aegisub 运行「自动化 → SekaiText → 从轴机拉取」，改动会带撤销点地原位替换进当前字幕。

同步宏无需手动安装：导出时若检测到本机装有 Aegisub，会自动装进其 `automation/autoload` 目录（首次使用重启一次 Aegisub 即可）；未检测到时宏文件会放在导出目录旁，复制进该目录一次即可。

## 成品样式与导出

导出默认就是成品：内置团队样式模板（1行 / 2行 / 3行 等定义）自动套用，按每行行数改样式、清理角色名与调试行。有自定义需求时，在「导出与同步选项」里选择自己的模板 `.ass` 即可覆盖内置模板。

## 一键压制

打轴完成后可直接压制（在页面下方）：内置 FFmpeg + libass，将字幕烧录进视频输出成品文件。

- 支持硬件加速，实测压制速度可提升 4～7 倍；
- 随引擎附带字幕默认字体，未装字体的机器也能压出正确字形。

## 常见问题

- **识别很慢**：识别为逐帧分析，时长与视频长度、机器性能相关，属正常现象；
- **行列表提示需要更新主程序**：插件是新版但 SekaiText 本体还是旧版，升级主程序到 5.2.0 以上即可；
- **导出的 ass 样式和团队成品不一致**：确认没有在导出选项里误选了别的模板文件（留空即用内置模板）。
